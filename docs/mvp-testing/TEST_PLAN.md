# Analytika MVP test plan

Manual acceptance script for the MVP. Upload the fixture, ask the questions in
order, and check the **Expected behavior** column. The LLM is non-deterministic,
so assert on *shape and behavior* (right regime, right test, a chart appears,
graceful failure) — **not** on exact wording.

Fixtures (generated by `generate_fixtures.py`, seed 42):
| File | Character | Purpose |
|---|---|---|
| `students_survey.csv` | 90 rows, clean, well-typed | Happy path — every capability should work |
| `sales_messy.csv` | 55 rows, dirty types/values | Recoverable errors — the repair loop should save it |
| `edge_cases.csv` | 12 rows, degenerate/hostile | Failure modes — should fail *gracefully*, not crash |
| `meta_extraction_macroman.csv` | 6 rows, Mac Roman + merged-cell band | Ingestion — encoding detection and header-band stripping |

Baked-in truths in `students_survey.csv` (so tests find real effects):
`study_hours_per_week` ↑ `exam_score`, `attendance_rate` ↑ `exam_score`,
`stress_level=High` ↓ `exam_score`, Female slightly higher, part-time job slightly lower.

Legend for regimes: **advisory**
(which test should I use), **pedagogy** (explain a concept), **exploratory**
(quick look + charts), **confirmatory** (formal stat test), **meta** (routing/
guardrails).

## B. Recoverable errors — `sales_messy.csv`

The point: these *should* fail on first code attempt and then be **auto-repaired**
(watch for the retry, or a clean final answer despite dirty data).

| # | You type | Expected behavior | Validates |
|---|---|---|---|
| B1 | `What's the total revenue?` | `revenue` is `"$1,234.56"` strings → first run may throw `could not convert string to float`; repair strips `$`/`,` and returns a number | Currency parse + repair-loop hint |
| B2 | `Average revenue by region` | Must first **normalize region** (`"North"`, `"north"`, `"North "`, `" South"`, `"SOUTH"` → 2 groups); returns a sensible grouped result | Categorical cleaning, whitespace/case |
| B3 | `Plot revenue over time` | `order_date` has 3 formats + blanks → parse with coercion, drop/flag bad dates, produce a time series | Mixed-date parsing, missing handling |
| B4 | `What percentage of orders were returned?` | `returned` is `Yes/No/Y/N/yes/no` → must map to a single boolean before counting | Messy boolean normalization |
| B5 | `Average customer age` | `customer_age` has ``""``/`"unknown"` → coerce to numeric, ignore invalid, report the mean + how many were dropped | Missing/invalid numeric coercion |
| B6 | `Is revenue higher in the North or South?` | Combines B1 + B2 cleaning, then a **t-test** on cleaned revenue by cleaned region | Cleaning + stats end-to-end |

**Watch for:** a repair that silently produces a *wrong* number (e.g., treating
`"N/A"` revenue as 0). The right behavior is to exclude invalid values and say so.

## C. Failure modes — `edge_cases.csv`

The point: the system should **degrade gracefully** — a clear message, not a raw
stack trace or an infinite repair loop.

| # | You type | Expected graceful behavior | Failure to watch for |
|---|---|---|---|
| C1 | `What's the average of Customer Name?` | Recognizes it's non-numeric; declines or explains — no crash | Type confusion |
| C2 | `Show total by Total $` | Column name has a space + `$` → must quote/reference it correctly (`df["Total $"]`), not `KeyError` | **KeyError** on odd column names |
| C3 | `Correlate constant_col with score` | `constant_col` has zero variance → correlation is undefined; should say so, not return NaN silently | Degenerate stats |
| C4 | `Analyze the all_missing column` | Column is entirely empty → clear "no data in this column" message | Empty-column handling |
| C5 | `Run a t-test on score by region` | Only 12 rows across 4 regions → some groups n=1–2; should **warn about tiny samples**, still respond | Small-n robustness |
| C6 | `What's the max score?` | `$1,999,999` outlier row should surface; answer is correct but ideally flags the outlier | Outlier awareness |
| C7 | `Are there duplicate customers?` | Should detect the duplicate "Ada Lovelace" row | Dup detection |

## E. Ingestion — `meta_extraction_macroman.csv`

Modelled on the real upload that exposed these bugs. The file is Mac Roman with a
merged-cell section band above the real header — the combination that used to
produce silent mojibake and 8 columns named `Unnamed: N`.

| # | You type | Expected behavior | Failure to watch for |
|---|---|---|---|
| E1 | *(just upload)* | Greeting reports **6 rows × 8 columns**, plus two ingest notes: encoding inferred as `mac_roman`, and 1 merged-cell row dropped | 7 rows, or no notes shown (frontend not deployed) |
| E2 | `What columns do I have?` | `Study ID`, `First author`, `Year`, `Analysed N`, `Age (summary)`, `Deaths n`, `Deaths N`, `Notes` | Any `Unnamed: N` or `Identification` — band row still being used as the header |
| E3 | `Show me the notes column` | Dashes render as `–` and `—`, comparisons as `≤` / `≥` | `Ð`, `Ñ`, `²`, `³` — decoded as latin-1 |
| E4 | `How much missing data is there?` | `Deaths n` / `Deaths N` reported as **~33% missing** (2 of 6 are `NR`) | "0% missing" — sentinels counted as present |
| E5 | `Summarise the mortality data`, then `explain the above results` | The follow-up explains **that answer**, carrying over any caveat | A dataset schema dump instead |

## D. System behaviors & guardrails (any dataset)

| # | You type | Expected behavior | Validates |
|---|---|---|---|
| D1 | `What's the weather today?` | Politely refuses — "I'm focused on data analysis…" | Off-topic guard (`is_off_topic`) |

| D6 | *(exhaust the free plan's 10 analyses)* | Metered runs blocked with an upgrade/limit message | Usage metering + gating |
| D7 | *(open the app in a second tab, upload a different CSV)* | Two independent **tasks** in the sidebar; switching loads each one's own history + artifacts | Multi-task persistence |
